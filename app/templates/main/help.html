Getting Started

1. Create a New Case

Start by creating a new investigation case from the Cases menu. Provide a descriptive name, case number, and investigator details.

2. Upload Artifacts

Upload JSON artifact files collected from Linux systems. LITE supports various artifact types including system logs, user data, and network information.

3. Monitor Ingestion

Track the ingestion progress on the dashboard. The system will automatically process and categorize your artifacts.

4. Analyze Data

Use the analysis tools to explore your data, create visualizations, and export findings.

  • Active - Case is currently being investigated
  • Inactive - Case is paused or on hold
  • Closed - Investigation completed
Case Organization

Each case maintains its own isolated database schema, ensuring data separation and integrity between investigations.

Artifact Ingestion

Supported File Types

LITE currently supports JSON format artifacts from Linux forensic tools.

Upload Process
  1. Select a case from the Cases list
  2. Click "Upload Artifacts" button
  3. Choose JSON files from your system
  4. Monitor ingestion progress in real-time
Artifact Categories
  • User Accounts & Groups
  • System Processes
  • Network Connections
  • File System Information
  • System Logs
  • Browser History
  • Security Events
  • Configuration Files
Ingestion Status
  • Processing - File is being ingested
  • Completed - Successfully processed
  • Failed - Error during processing

Troubleshooting

Common Issues

  • Ensure file is in valid JSON format
  • Check file size limits
  • Verify case is in active status
  • Check disk space availability

  • Check PostgreSQL service status
  • Verify database credentials
  • Ensure database server is accessible
  • Check network connectivity

  • Monitor system resources (CPU, Memory, Disk)
  • Check database performance and indexing
  • Consider archiving old cases
  • Optimize large file processing
Getting Support

For additional support or to report issues:

  • Check the system health status at /health
  • Review application logs for error details
  • Contact your system administrator
